## Already Built (packages/ui)

### Button
**File:** `Button.vue`
**Variants:** `primary` | `secondary` | `ghost` | `destructive`
**Sizes:** `sm` | `md` | `lg`
**States:** default | hover | focus | disabled | loading (spinner replaces label)
**Used on:** every page — primary actions (Save, Approve, Block), secondary (Cancel, Export CSV), ghost (Edit, Re-assign), destructive (Delete, Block Account)

**File:** `Badge.vue`

## Needs to Be Built (add to packages/ui before use)

### DataTable
**Purpose:** Full-width TanStack Table v8 wrapper. The only table component — never build ad-hoc tables.
**Props:** columns (ColumnDef[]), data, loading, pagination, onRowClick
**Features required:**
- Full viewport width (no horizontal scroll on desktop)
- Row click → fires onRowClick handler (caller handles navigation)
- Right-click row → browser native "open in new tab" (use `<a>` wrapper on row)
- Column-level sort (click header)
- All cells top-aligned; text left-aligned; numbers right-aligned
- Auto row height — never clips content, never truncates with ellipsis
- Loading state: skeleton rows (8 rows placeholder)
- Empty state: renders EmptyState component
- "N new items" live-update banner (subtle pulse at top, for real-time tables)
**States:** loading | empty | populated | live-updating
**Used on:** every table page — Customers, Transactions, KYC Queue, Risk Queue, B2B Clients, Ledger Entries, Partner Commissions, System Events, all sub-tables in entity pages

### FilterBar
**Purpose:** Filter row above DataTable. Contains search input + dropdown filters + date range picker + active filter chips.
**Props:** filters (FilterConfig[]), onChange, onReset
**Features:**
- Search input (text, searches server-side)
- Each dropdown filter has a search input inside (FE search in dropdowns)
- Active filters shown as dismissible chips
- Filter state persisted in URL query params
- "Reset filters" clears all to default
**States:** collapsed (search only) | expanded (all filters) | has-active-filters (chips visible)
**Used on:** all table pages

### PageHeader
**Purpose:** Top of every page. Breadcrumb left, action buttons right, admin info right-most.
**Props:** breadcrumbs (array of {label, href}), actions (slot), adminName, lastLogin
**Breadcrumb rules:**
- Each prior level is a clickable link
- Current page is plain text (not a link)
- Format: `Customers › usr_001, Alice Demo › Activity`
**Used on:** every page

### SidePanel
**Purpose:** Secondary actions panel. Pushes main content left — never overlays. Closes via URL change.
**Props:** open, width (480px | 560px token), title, onClose
**URL behavior:** Opens with query param or sub-route (e.g. `?panel=add-note`). Closing returns to parent URL.
**Width:** 480px default, 560px for complex forms
**States:** closed | open | loading (skeleton content)
**Used on:** all entity detail pages — Add Note, Change Status, Approve/Reject, Re-assign, Freeze Amount, Block Account, Edit Personal Info
**Rule:** No modals anywhere. No drawers that overlay. This is the ONLY secondary action pattern.

### DefinitionList
**Purpose:** Label-above-value pairs for entity detail sections. Replaces info cards / data cards from legacy Figma.
**Props:** items (array of {label, value, subtext?, href?, type: 'text'|'money'|'status'|'link'|'badge'})
**States per item:**
- `null/undefined` → render `—` (em dash)
- `0` → render `0` (not dash — data exists)
- value is a link → render as `<a>` (clickable, right-click to new tab)
- value is money → use MoneyAmount component
- value is status → use StatusPill
- long text → wraps fully, never truncated
- subtext → smaller text below value (e.g., "Updated 13 Aug 2024 by Vlad")
**Used on:** all entity detail pages (Customer, Transaction, B2B Client, Widget Config, Risk Review)

### InfoSection
**Purpose:** A labeled group of DefinitionList items with optional Edit button and action buttons.
**Props:** title, items, editLabel?, onEdit?, actions (slot)
**States:** view | edit (fields become inputs) | saving
**Used on:** Customer detail — Personal Info, KYC, Rewards, Wallets sections

### StatCard
**Purpose:** Single KPI card for Dashboard.
**Props:** label, value, subvalue?, delta?, deltaDirection: 'up'|'down'|'neutral', trend (optional sparkline data)
**States:** loading (skeleton) | populated
**Used on:** Dashboard only — customer counts, transaction totals, KYC queue depth, risk queue depth

### RiskScore
**Purpose:** Visual risk score display.
**Props:** score (0-100), level: 'low'|'medium'|'high'|'critical'
**Renders:** Score number + level label + color bar
**Color mapping:** low=success, medium=warning, high=error, critical=error-dark
**Used on:** Customer detail header, Transaction detail, Risk Review entity, Transactions table column

### MoneyAmount
**Purpose:** Displays a monetary amount with correct decimal places and currency.
**Props:** amount (string|number), currency, showUSD? (secondary USD equivalent), size: 'sm'|'md'|'lg'
**Rules:** Right-aligned always. Never rounds without explicit prop. Shows 8 decimal places for crypto.
**Used on:** all tables (Amount column), all entity pages (fee breakdowns, balances, commissions)

### CurrencyIcon
**Purpose:** Maps currency code to icon + color. Fallback: code letters in a circle.
**Props:** currency (string), size: 'xs'|'sm'|'md'
**Currencies:** EUR, USD, GBP, BTC, ETH, USDT + generic fallback
**Used on:** Transactions table (Currency column), Customer wallets, Ledger Entries

### LiveIndicator
**Purpose:** Shows when a table has real-time updates active.
**Props:** active (boolean), label? ("Live" default)
**Renders:** green pulsing dot + label
**Used on:** Transactions table, KYC Queue, Risk Queue, System Events table

### SearchInput
**Purpose:** Full-text search input, debounced, server-side.
**Props:** placeholder, onSearch, debounceMs (300 default)
**Used on:** FilterBar (embedded), standalone on all table pages

### DateRangePicker
**Purpose:** From/to date picker for table filters.
**Props:** from, to, onChange, presets (Last 7d, Last 30d, Last 90d, Custom)
**Used on:** FilterBar on Transactions, Customers, Ledger Entries, Partner Commissions, System Events

### ConfirmPanel
**Purpose:** A SidePanel variant specifically for confirm-action flows (approve, reject, block, freeze). Replaces the modals shown in legacy Figma.
**Props:** title, description, fields (read-only entity info), reasonRequired (boolean), actionLabel, actionVariant: 'primary'|'destructive', onConfirm, onCancel
**Fields always shown (read-only):** entity name, entity ID, action being taken
**Reason field:** required textarea when reasonRequired=true
**States:** idle | submitting | error
**Used on:**
- Block Customer Account
- Activate Customer Account
- Approve Transaction
- Reject Transaction
- Freeze Wallet Amount
- Block Wallet
- Approve KYC
- Reject KYC
- Approve Risk Review
- Reject Risk Review
- Delete Staff User

### Breadcrumb
**Purpose:** Navigation trail in PageHeader.
**Props:** items (array of {label, href?})
**Rule:** Last item has no href (current page). All others are `<a>` elements.
**Used on:** all pages via PageHeader

### Tabs (FilterTabs)
**Purpose:** Tab switcher that changes table filter, NOT page content. Changes URL query param.
**Note:** Per ADMIN_UX_RULES no tabs inside entity pages. FilterTabs are only for table-level type filters (e.g. All | Fiat | Crypto).
**Props:** tabs (array of {label, value, count?}), active, onChange
**Used on:** Transactions table (All / Fiat / Crypto type filter)

### AdminShell
**Purpose:** Top-level layout: collapsible left sidebar + main content area.
**Props:** none (reads route)
**Sidebar behavior:**
- Open state: icon + label
- Collapsed state: icon only
- Expandable groups (Compliance, Settings) — accordion does NOT auto-close other groups
- Active item highlighted (no yellow background — use left border + text color)
**Used on:** wraps the entire admin app
